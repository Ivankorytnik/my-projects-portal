# Мой портал проектов

Готовый статический портал для публикации на GitHub Pages.

## Что есть в портале

- вход по паролю;
- карточки проектов;
- разделение на личные и рабочие проекты;
- поиск;
- фильтры по статусу и категории;
- добавление, изменение и удаление проектов;
- сохранение данных в браузере;
- экспорт и импорт резервной копии в JSON;
- адаптация под телефон и компьютер.

## Как изменить пароль

1. Придумайте новый пароль.
2. Получите SHA-256 хеш пароля, например через любой локальный SHA-256 генератор.
3. Откройте файл `app.js`.
4. В первой строке замените значение `PASSWORD_HASH`.

Важно: GitHub Pages является статическим хостингом. Такой пароль скрывает интерфейс от обычного посетителя, но технически подкованный человек может изучить код сайта. Для настоящей защиты данных используйте Firebase Authentication, Supabase Auth, Cloudflare Access или серверную авторизацию.

## Как опубликовать на GitHub Pages

1. Создайте новый репозиторий, например `my-portal`.
2. Загрузите в корень репозитория файлы:
   - `index.html`
   - `style.css`
   - `app.js`
3. Откройте `Settings` → `Pages`.
4. В разделе `Build and deployment` выберите:
   - Source: `Deploy from a branch`
   - Branch: `main`
   - Folder: `/root`
5. Нажмите `Save`.
6. Через несколько минут портал будет доступен по адресу:
   `https://ВАШ_ЛОГИН.github.io/my-portal/`

## Где хранятся проекты

Добавленные через интерфейс проекты сохраняются в `localStorage` конкретного браузера.

Чтобы не потерять данные:
- периодически нажимайте «Экспорт проектов»;
- храните файл `projects-backup.json`;
- на другом компьютере используйте «Импорт проектов».

## Настройка стартовых проектов

Стартовые карточки находятся в массиве `initialProjects` внутри файла `app.js`.
